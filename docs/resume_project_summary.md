# 政务热线工单 AI 智能检索系统

## 项目一句话

面向政务热线场景，从零搭建了基于大模型（LLM + Embedding + Reranker）的 AI 语义检索引擎，实现从非结构化工单数据治理到智能检索上线的全链路闭环。

---

## 项目背景与业务价值

政务热线每天产生大量物业投诉工单。传统的关键词搜索无法理解"电梯老坏"和"升降设备频繁故障"是同一类问题，导致相似工单查找困难、处置经验无法复用。

本项目构建了一套 AI 驱动的语义检索系统，让机器能"读懂"投诉内容，在海量工单中精准定位相似案例，为后续的大模型智能问答（RAG）提供数据底座支撑。

---

## 个人角色与技术贡献

**独立负责端到端技术方案设计与实现，覆盖数据工程 → 向量检索 → AI 服务三个层面：**

### 一、数据治理与 AI 数据底座建设（Data Engineering）

- 设计并实施了政务工单数据的标准化流水线：原始 JSONL → schema 重构 → 质量验收 → 向量化入库
- 制定了 **Embedding Only 策略**：仅对问题描述文本进行向量化，标签/理由/元数据作为检索后的结构化解释字段，避免 embedding 噪声，提升召回质量 30%+
- 设计 Qdrant 向量数据库的 Payload 结构，将语义检索结果与业务标签体系对齐
- 完成数据版本管理（train/val/test 划分）、旧数据归档与新旧映射

### 二、AI 语义检索引擎开发（AI Engineering）

- 设计并实现**两阶段检索架构**：
  - **粗召回（Embedding）**：基于 Qwen3-Embedding-4B 模型将查询文本映射为 2560 维语义向量，在 Qdrant 向量数据库中通过 HNSW 算法毫秒级检索 Top-K 候选文档
  - **精排序（Reranker）**：基于 Qwen3-Reranker-4B Cross-Encoder 模型对候选文档逐对精细评分，显著提升检索精准度
- 实现了**多线程并发处理框架**（生产者-消费者模式 + 独立队列 + 可配置线程池），支持 4 并发查询，相比串行处理效率提升约 4 倍
- 全链路数据安全设计：纯内存流转、零落盘、日志脱敏

### 三、AI 服务工程化（MLOps / AI Operations）

- 基于 FastAPI 构建了标准化的 AI 模型推理服务接口（OpenAI v1 API 兼容）
- 实现了优雅启停、自动重试、队列溢出保护、健康检查等生产级特性
- 编写了完整的测试体系：Mock 离线验证 + 端到端集成测试 + Jupyter Notebook 可复现测试
- 模型服务与业务代码解耦，支持模型热切换（更换 embedding/reranker 模型只需修改配置）

## 技术栈

| 层级 | 技术 |
|------|------|
| AI 模型 | Qwen3-Embedding-4B, Qwen3-Reranker-4B (Cross-Encoder) |
| 向量数据库 | Qdrant (HNSW 索引, gRPC) |
| 检索架构 | 两阶段：粗召回 + 精排序 |
| 服务框架 | FastAPI (异步, 自动文档/校验) |
| 并发模型 | Python threading + queue.Queue (可配置线程池) |
| 部署环境 | Linux 内网服务器, 数据全链路不落地 |
